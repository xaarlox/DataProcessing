# Аналіз сезонності продажів
Проєкт присвячений розвідувальному аналізу даних (EDA) щодо продажів у різних містах. За допомогою Python та бібліотеки Pandas скрипт обробляє історичні дані, будує зведені таблиці та виявляє міста з найбільш нестабільним попитом, спираючись на статистичні показники дисперсії.

## Постановка задачі
Побудувати `pivot_table` «місяць × місто» для суми total, знайти місто з найбільшою сезонною різницею (max-min) та пояснити результат через статистики.

**Примітка**: Для аналізу використовується датасет [*Superstore Sales Forecasting*](https://www.kaggle.com/datasets/rohitsahoo/sales-forecasting/data).

## Хід виконання роботи
1. *Підготовка даних*: Зчитування CSV-файлу (`utf-8`). Конвертація колонки `Order Date` у формат `datetime` з явним вказуванням європейського формату (`dayfirst=True`). Витягнення номера місяця.
2. *Побудова зведеної таблиці*: Створення `pivot_table`, де рядками виступають міста, колонками - місяці, а значеннями - агрегована сума продажів.
3. *Пошук сезонної різниці*: Обчислення розмаху для кожного міста по горизонтальній осі (`axis=1`). Визначення міста з найбільшою амплітудою коливань.
4. *Фільтрація*: Підрахунок загальної суми продажів по кожному місту для виведення на екран лише ТОП-10 міст.
5. *Статистичний аналіз*: Обчислення базових статистик для міста з найбільшою різницею - середнього значення продажів на місяць та стандартного відхилення, яке показує середній рівень відхилення попиту від норми.
6. *Візуалізація*: Побудова лінійного графіка сезонності за допомогою `matplotlib` та `seaborn`.

## Особливості реалізації
+ Використано `.fillna(0)`, оскільки не в усіх містах були продажі кожного з 12 місяців.
+ Для кращого візуального сприйняття таблиці дані округлено до цілих чисел `.round(0).astype(int)`.
+ На графіку динамічно розраховуються та автоматично підсвічуються екстремуми: зеленим маркером виділено пік сезону, а червоним - найбільший спад.
